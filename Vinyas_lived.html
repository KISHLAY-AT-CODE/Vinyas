Vinyas: The Journey

A Gamified Syllabus Tracker, Real-Time Chrome Extension Companion, and AI-Powered Study Planner. This is the story of how we revolutionized daily preparation for competitive exams.

The Spark & The Genesis

Preparing for competitive examinations in India, such as JEE, NEET, or BITSAT, is a monumental test of endurance. Students spend upwards of twelve hours daily immersed in video lectures, solving infinite sheets of Daily Practice Problems (DPPs), and tracking their errors in massive, physically heavy study booklets. In this intense process, traditional tracking utilities fail. Keeping spreadsheets up to date becomes a chore, and studying in isolation leads to burnout.

"Passive watching creates an illusion of competence. True learning only begins when you close the video player, open the question module, and grapple with a challenging problem. Vinyas was built to celebrate that exact shift."

Vinyas was conceptualized as a personal study hub that removes friction. The core vision was simple: a student studying on a digital platform like PhysicsWallah (PW) should not have to manually log their work. An intelligent Chrome extension companion would detect when they are watching lectures or completing quizzes, capturing their metrics programmatically and feeding them to a serverless MongoDB cloud backend.

This basic idea evolved into a comprehensive gamified tracker. We added XP matrices, Pomodoro timers, flashcards, customized daily planners, and multi-textbook support, turning a solitary grind into a highly engaging, visually rewarding game.

Features in Focus

An in-depth look at the modules that form the Vinyas ecosystem.

🧩

The Chrome Companion

A Manifest V3 extension that pairs with the dashboard via a 1-click Auto-Pair Sync ID. It injects a draggable, bounded glassmorphic widget directly into textbook exercise portals. Students label question states (Easy, Medium, Hard, Difficult) on-page, and the extension proxy submits their answers to the backend, completely eliminating split-screen setups.

🏆

Gamified Progress

Every action in Vinyas earns Experience Points (XP). The dashboard tracks study streaks using a visual activity heatmap. Maintaining a streak activates XP multipliers (1.2x for 3 days, 1.5x for 7 days), driving student engagement. Reaching milestones triggers a full-screen animated celebration overlay.

🧠

Syllabus Auto-Builder

Generating an exam syllabus is instant with server-hosted templates (BITSAT, JEE, NEET). Students skip manual setup, choosing an exam template and adjusting chapters or importing custom textbook modules containing detailed exercise tracking configurations.

⏱️

Focus & Revisions

A built-in Pomodoro Timer logs focus blocks to specific chapters, awarding XP and sounding A5 audio beeps on completion. For retention, a Spaced Repetition card deck schedules chapters for review based on recall difficulty (Again: 1 day, Hard: 3 days, Good: 7 days, Easy: 14 days).

🤖

AI Study Companion

A conversational Study Buddy helps optimize prep. It analyzes syllabus progress, daily checklists, and mock test scores. Using a rotating load-balancer of up to 20 Google Gemini keys, with seamless failovers to Cerebras and Groq, it provides instant advice and productivity plans.

🛡️

Encrypted Backups

Protecting user data is key. Weekly automated cron tasks bundle a student's database state, encrypting it client-side with AES-256-GCM (using PBKDF2 with 100,000 iterations) using their Sync ID. This secure file is emailed to their inbox, meaning even if their email is compromised, the data remains unreadable.

Engineering Challenges & Victories

Developing Vinyas required addressing significant technical challenges. A key issue was **State Collisions & Ghost Overwrites**. Early versions cached progress in browser `localStorage` and synced with MongoDB asynchronously. In multi-tab environments, this created race conditions where older cached states would overwrite newer database records. The solution was removing local storage overrides entirely and establishing the MongoDB cloud database as the absolute single source of truth, with live tab communication managed via custom background events.

Another major hurdle was **Dynamic SPA Interception**. The PhysicsWallah platform operates as a React Single Page Application (SPA), meaning conventional URL matching failed since the page loads content dynamically. We resolved this by refactoring content scripts to run lightweight, high-frequency `MutationObserver` loops alongside targeted DOM polling. This allowed the extension to detect quiz elements and inject the tracker widget reliably without impacting host performance.

Finally, configuring the **AI study buddy** required managing strict API rate limits. We built a robust serverless API gateway that cycles through 20 Google Gemini API keys. In case of key exhaustion, the gateway falls back to Cerebras and Groq in milliseconds, maintaining a seamless experience for the user.
